# Active Directory and Windows Server Forensics

## Overview

This project demonstrates forensic analysis of a Windows Server environment and Active Directory infrastructure. The investigation focused on identifying domain configuration details, user accounts, security groups, organizational structures, and system information relevant to enterprise forensic investigations.

The objective was to collect and analyze Active Directory artifacts to better understand user activity, administrative privileges, domain membership, and overall network configuration within a Windows domain environment.

---

## Objectives

- Identify Active Directory domain information
- Enumerate domain users and security groups
- Review administrative privileges and memberships
- Analyze Windows Server configuration details
- Document Active Directory forensic artifacts
- Identify potential indicators of unauthorized access or privilege escalation

---

## Tools Used

### Microsoft Windows Server
Used to examine domain services, user accounts, and Active Directory objects.

### PowerShell
Used to query Active Directory information and collect forensic artifacts.

### Active Directory Users and Computers (ADUC)
Used to review domain objects and administrative relationships.

### Command Line Utilities
Used to enumerate users, groups, computers, and system information.

## Security Considerations

Active Directory environments are critical infrastructure components within enterprise networks. Improper privilege management, excessive administrative permissions, and unauthorized account creation can significantly increase organizational risk.

Regular auditing of users, groups, and administrative privileges is essential to maintaining a secure domain environment.
